title: Codegen & the CLI
description: The convex-localfirst CLI — codegen derives the client manifest from your DSL, check catches illegal writes, dev runs both.
---

The `@convex-localfirst/cli` turns your server DSL into the client manifest and
guards your codebase against writes that would bypass local-first.

Because it introspects your TypeScript Convex modules, run it under a TS loader:

```bash
node --import tsx node_modules/@convex-localfirst/cli/dist/index.js <command>
```

Add scripts so you don't retype that:

```json title="package.json"
{
  "scripts": {
    "generate": "node --import tsx node_modules/@convex-localfirst/cli/dist/index.js codegen",
    "check": "node --import tsx node_modules/@convex-localfirst/cli/dist/index.js check"
  }
}
```

## `codegen`

Reads every `lf.table(...).query/insert/patch/remove` in your Convex modules and
emits a **declarative manifest** the client engine runs locally. It parses the
`value` / `patch` / `id` / `key` closures, derives each query's pull **scope**
(`byUser`, and `byWorkspace` / `byProject` from the scope's id field), and writes:

```
src/convex-localfirst/generated.ts   →  export const localFirstManifest = { ... }
```

Re-run it whenever you add or change a local-first function. Import the result in
your provider:

```ts
import { localFirstManifest } from "./convex-localfirst/generated";
```

<Callout>
Modules that can only run inside the Convex runtime (e.g. your `sync.ts`, which
imports `_generated`) are **skipped with a warning** rather than failing codegen.
Only `lf.table` declarations are introspected.
</Callout>

## `check`

Static analysis that fails if any Convex function writes **directly** to a
local-first table with `ctx.db.insert` / `ctx.db.replace` — those writes would
skip the sync ledger and change log, corrupting the local-first contract
([invariant I10](/docs/reference/invariants)). Run it in CI:

```bash
npm run check
# convex-localfirst check: no direct writes to local-first tables found
```

If it finds violations it prints each file and exits non-zero.

## `dev`

Runs the dev-time pipeline — `codegen` then `check` — so your manifest is fresh
and your writes are legal:

```bash
node --import tsx node_modules/@convex-localfirst/cli/dist/index.js dev
```

Run the Convex backend itself separately with `npx convex dev`.

## `init`

Scaffolds the starting files for a new local-first setup.
