This walks through a complete, working setup: a per-user `todos` table that reads
and writes locally, persists offline, and syncs to Convex. Every step is exactly
what the [example app](/docs/reference/example) does.

Each table is declared **once**, in one module. The Convex schema derives from it,
the server sync config derives from it, and the client runs the very same
declaration locally — there is no codegen step and nothing to keep in sync.

<Callout title="Prerequisites">
A Convex project (`npm create convex@latest` or an existing one) and React. You
should already have `convex/` and `convex/_generated/` from running
`npx convex dev` at least once.
</Callout>

## 1. Install

```bash
npm install convex-localfirst
```

One package: the client surface is the root import, the DSL is
`convex-localfirst/server`, the mountable component ships inside it, and the
CLI comes with it — `npx convex-localfirst init` scaffolds every file below,
and `npx convex-localfirst check` is a CI guard against illegal direct writes.

## 2. Mount the component

The component provides all sync bookkeeping (ledger, change log, id map,
row versions) so you never hand-write those tables. Mount it in
`convex/convex.config.ts`:

```ts title="convex/convex.config.ts"
import { defineApp } from "convex/server";
import localfirst from "convex-localfirst/component/convex.config.js";

const app = defineApp();
app.use(localfirst);

export default app;
```

## 3. Declare the table — once

Create the `lf` factory, then declare the table's **shape, scope, and indexes**
together with the functions your React code calls:

```ts title="convex/localfirst.ts"
import { createLocalFirst } from "convex-localfirst/server";

export const lf = createLocalFirst();
```

```ts title="convex/todos.ts"
import { v } from "convex/values";
import { lf } from "./localfirst";

export const todos = lf.table("todos", {
  shape: {
    ownerId: v.string(),
    listId: v.string(),
    text: v.string(),
    done: v.boolean(),
    createdAt: v.number(),
    updatedAt: v.number()
  },
  scope: lf.byUser("ownerId"),       // per-user isolation, enforced on the server
  indexes: { byList: ["ownerId", "listId", "createdAt"] }
});

export const list = todos.query({
  args: { listId: v.string() },
  index: "byList",
  key: ({ auth, args }) => [auth.userId, args.listId],
  order: "asc",
  initial: []
});

export const create = todos.insert({
  args: { listId: v.string(), text: v.string() },
  value: ({ auth, args, now }) => ({
    ownerId: auth.userId,
    listId: args.listId,
    text: args.text,
    done: false,
    createdAt: now,
    updatedAt: now
  })
});

export const toggle = todos.patch({
  args: { id: v.string(), done: v.boolean() },
  patch: ({ args, now }) => ({ done: args.done, updatedAt: now })
});

export const remove = todos.remove({ args: { id: v.string() } });
```

The `value` / `patch` / `key` closures run **on the client** (optimistically,
offline) and the writes are re-validated on the server — keep them pure
(`auth`, `args`, `now`, `localId`; `ctx` throws). See
[Defining local-first tables](/docs/defining-tables) for scopes, conflict
policies, and the full spec shapes.

## 4. Derive the schema and the sync endpoints

Both are one-liners over the modules you just wrote — nothing is restated:

```ts title="convex/schema.ts"
import { defineSchema } from "convex/server";
import { todos } from "./todos";

export default defineSchema({
  todos: todos.table()   // adds the localId id field + the declared indexes
});
```

```ts title="convex/sync.ts"
import { collectTables, createSyncFunctions } from "convex-localfirst/server";
import { components } from "./_generated/api";
import { mutation, query } from "./_generated/server";
import * as todos from "./todos";

export const { push, pull } = createSyncFunctions({
  component: components.convexLocalFirst,
  mutation,
  query,
  tables: collectTables({ todos })
  // For byWorkspace / byProject tables, add an `isMember` callback — see Syncing.
});
```

Add a new local-first table by adding its module to both imports. Then deploy:

```bash
npx convex dev
```

## 5. Wire the provider

The client imports the **same table modules** and hands them to the drop-in
`ConvexProvider`. IndexedDB persistence, the sync transport
(`api.sync.push` / `api.sync.pull`), and a client id are all defaulted:

```tsx title="src/main.tsx"
import ReactDOM from "react-dom/client";
import { ConvexProvider, ConvexReactClient } from "convex-localfirst";
import * as todos from "../convex/todos";
import { App } from "./App";

const userId = "demo-user";                       // from your auth provider in production
const convex = new ConvexReactClient(import.meta.env.VITE_CONVEX_URL);

ReactDOM.createRoot(document.getElementById("root")!).render(
  <ConvexProvider client={convex} localFirst={{ modules: { todos }, userId }}>
    <App />
  </ConvexProvider>
);
```

<Callout title="How this works">
`lf.table` attaches each declaration (including the real closures) to its
exports. In the browser those exports are metadata-only stubs — no Convex
function is registered client-side — and the provider assembles the local
manifest from them at runtime. Key `modules` by the Convex module path:
`{ todos }` → `api.todos.*`, a nested module as `{ "tasks/todos": todos }`.
</Callout>

<Callout type="warn" title="Per-user local data">
The default IndexedDB store is namespaced by `userId`, so one user's local data
is isolated from another's on a shared device, and switching users on logout
switches stores. See [invariant I9](/docs/reference/invariants).
</Callout>

## 6. Use the hooks

Exactly the Convex hooks — declared functions are now local-first:

```tsx title="src/App.tsx"
import { useMutation, useQuery, useSyncStatus } from "convex-localfirst";
import { api } from "../convex/_generated/api";

export function App() {
  const todos = useQuery(api.todos.list, { listId: "inbox" }, { initial: [] }) ?? [];
  const create = useMutation(api.todos.create);
  const toggle = useMutation(api.todos.toggle);
  const sync = useSyncStatus();

  return (
    <main>
      <p>{sync.online ? "🟢" : "🔴"} · {sync.pendingMutations} pending</p>
      <button onClick={() => create({ listId: "inbox", text: "First todo" })}>Add</button>
      <ul>
        {todos.map((t) => (
          <li key={t._id} onClick={() => toggle({ id: t._id, done: !t.done })}>
            {t.done ? "✅" : "⬜️"} {t.text}
          </li>
        ))}
      </ul>
    </main>
  );
}
```

Writes commit locally and update the UI immediately, then sync. Reload the page
offline and your data is still there. Next: [How it works](/docs/concepts).
