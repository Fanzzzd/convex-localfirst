Everything below is public. Engine internals (rebase, the engine class, the store
implementation) are intentionally not exported; they live behind
`convex-localfirst/core/internal` for the React adapter only.

## `convex-localfirst` — the React client

### `ConvexProvider`

```ts
function ConvexProvider(props: {
  client: ConvexReactClient;
  children: React.ReactNode;
  localFirst?: LocalFirstProviderConfig;
}): JSX.Element
```

Drop-in for Convex's provider. Without `localFirst`, behaves exactly like
`convex/react`. See [the provider](/docs/react-hooks#the-provider).

```ts
type LocalFirstProviderConfig = {
  modules?: Record<string, unknown>;   // imported lf.table modules (runtime manifest, no codegen)
  manifest?: LocalFirstManifest;       // escape hatch / tests, instead of modules
  schemaVersion?: number;              // gates sync on incompatible shape changes; default 1
  userId?: string | null;
  clientId?: string;
  store?: LocalStore;                  // default: IndexedDB in the browser, in-memory elsewhere
  databaseName?: string;               // default "convex-localfirst"
  namespace?: string;                  // default userId (device isolation, I9)
  sync?: {
    push?: FunctionReference<"mutation">;   // default api.sync.push
    pull?: FunctionReference<"query">;      // default api.sync.pull
    presence?: FunctionReference<"mutation">;   // default api.sync.presence
    presenceList?: FunctionReference<"query">;  // default api.sync.presenceList
  };
  transport?: SyncTransport;           // escape hatch: bring your own
  nameOf?: FunctionNameResolver;       // defaults to Convex getFunctionName
  attachments?: {                      // offline attachments; default attachments:getUploadUrl/finalize
    getUploadUrl?: FunctionReference<"mutation">;
    finalize?: FunctionReference<"mutation">;
    storageIdField?: string;           // default "storageId"
    backend?: AttachmentBackend;       // escape hatch: your own upload transport
  };
};
```

### `useQuery` / `useMutation`

Args and result infer from the Convex function reference — no explicit generics.
Arg arity mirrors `convex/react`'s `OptionalRestArgs`: functions with required
args require them; empty-args functions may omit them.

```ts
function useQuery<Query extends FunctionReference<"query">>(
  reference: Query,
  ...argsAndOptions   // args?: FunctionArgs<Query> | "skip", options?: { initial?; sync?: "auto" | "off" }
): FunctionReturnType<Query> | undefined

function useMutation<Mutation extends FunctionReference<"mutation">>(
  reference: Mutation
): (...args: OptionalRestArgs<Mutation>) => LocalFirstMutationCall<FunctionReturnType<Mutation>>
```

A declared local-first reference is served by the engine; every other reference
falls through to Convex. See [React hooks](/docs/react-hooks).

### `LocalFirstMutationCall`

```ts
type LocalFirstMutationCall<T> = Promise<T> & {
  opId: string;
  id: LocalId;              // the canonical target row id, available SYNCHRONOUSLY (insert-then-patch seam)
  local: Promise<LocalCommit>;
  server: Promise<T>;
  status: () => MutationStatus;
};
```

### Typed queries

```ts
function createLocalDb<Modules>(modules: Modules): LocalDb<Modules>   // typed db root
function collection<Row>(table: string): LocalQuery<Row>             // untyped escape hatch

function useLiveQuery(plan | "skip", options?: { pollMs? }):
  Array<Row & Rel> | ReadonlyMap<string | null, Array<Row & Rel>> | undefined   // Map when grouped
function useLiveCounts(plan | "skip", options?: { pollMs? }):
  number | Record<string, number> | undefined                                    // Record when grouped
```

Query builder methods (on `db.<table>` and `collection(...)`):

| Method | Meaning |
| --- | --- |
| `.scope(values)` | Scope fields to read/pull (drives the background pull). |
| `.filter(spec)` | Serializable, planner-visible [`FilterSpec`](/docs/filters). AND-chains. |
| `.where(row => boolean)` | Closure escape hatch (opaque; evaluated post-index). |
| `.order(field, "asc" \| "desc")` | Sort by a field. |
| `.orderBy(field, dir)` | Alias of `.order` on the typed root. |
| `.limit(n)` | Cap the result count. |
| `.groupBy(field)` | Turn the result into a live `Map`/counts record. |
| `.with(...names)` | Attach [declared relations](/docs/typed-db) by name (typed root). |
| `.related(name, spec)` / `.withRelations(map)` | Per-query relation escape hatches. |

Relation combinators (per-query): `one`, `many`, `manyToMany`, `viaIds`.
Declared-relation descriptors on `lf.table`: `lf.one`, `lf.many`, `lf.backref`.

## `convex-localfirst/server`

### `createLocalFirst`

```ts
function createLocalFirst<Ctx>(options?: {
  defaults?: { idField?: string };
  schemaVersion?: number;
}): LfFactory
```

Identity is resolved server-side at sync time (in `createSyncFunctions`), not
here. Returns the `lf` factory:

| Member | Signature |
| --- | --- |
| `lf.table(name, options)` | see below → `{ table, query, insert, patch, remove }` |
| `lf.byUser(field)` | `ScopeDefinition` |
| `lf.byWorkspace({ workspaceIdField, membershipTable })` | `ScopeDefinition` |
| `lf.byProject({ projectIdField, membershipTable })` | `ScopeDefinition` |
| `lf.one(table, fk)` / `lf.many(table, { via })` / `lf.backref(table, fk)` | [declared relations](/docs/typed-db) |

`lf.table` options: `shape` (required validators), `scope` (required), `idField`
(default `"localId"`), `indexes`, `timestamps` (`true` or `[created, updated]`),
`relations`, `setFields`, `counterFields`, `searchFields`, `serverFields`,
`clientCan`. Builder specs (each optional — a bare call derives args from the shape):

```ts
table.query({  args, index, key, order?, initial? })
table.insert({ args, value })   // value: ({ ctx, auth, args, now, localId }) => Row
table.patch({  args, id?, patch? })
table.remove({ args?, id? })
```

### `collectTables` / `createSyncFunctions`

```ts
function collectTables(modules: Record<string, unknown>): SyncConfig["tables"]

function createSyncFunctions(options: {
  component: typeof components.convexLocalFirst;
  mutation: MutationBuilder;
  query: QueryBuilder;
  internalMutation?: InternalMutationBuilder;   // exposes `gc` as an internal cron target
  tables: SyncConfig["tables"];
  access?: {
    member: (ctx, { userId, scopeValue, table, membershipTable }) => Role | null | Promise<Role | null>;
    read?: (ctx, { userId, role, table, row }) => boolean | Promise<boolean>;
    write?: (ctx, { userId, role, table, action, before, patch?, proposed }) => boolean | Promise<boolean>;
  };
  getUserId?: (ctx) => string | null | Promise<string | null>;   // default: identity.tokenIdentifier
  devUnsafeAllowClientUserId?: boolean;                          // local demo only — never in prod
  schemaVersion?: number;
  now?: () => number;
  changeRetentionMs?: number;                                    // change-log GC horizon, default 30 days
  serverStamp?: Record<string, { fields: string[]; stamp: (ctx, { userId, value }) => Record<string, unknown> | undefined }>;
  onWrite?: (ctx, { table, action, before, after, userId, functionName }) => Promise<void>;
}): { push; pull; presence; presenceList; gc; serverWriter }
```

See [Syncing](/docs/syncing#createsyncfunctions) and [Permissions](/docs/permissions).

`serverWriter(ctx, userId?)` is the trusted server-side writer — call it from your
own mutations (activity fan-out, importers, crons) to write local-first tables
**through the change log**, so every client syncs the write:

```ts
const w = serverWriter(ctx);
const { localId } = await w.insert("activities", { workspaceId, verb: "created" });
await w.patch("issues", issueId, { sub_issues_count: 3 });
await w.remove("comments", commentId);
```

## `convex-localfirst/core`

Wiring helpers for a headless (non-hook) engine:

| Export | Role |
| --- | --- |
| `createLocalFirstEngine(options)` | Build an engine without React (service layer, worker, store). |
| `IndexedDbStore({ databaseName, namespace })` | Browser persistence (`namespace` isolates per-user data, I9). |
| `MemoryLocalStore` | In-memory `LocalStore` — the default in tests/SSR. |
| `createConvexTransport({ client, push, pull, clientId, userId })` | The default sync transport. |
| `createClientId()` | A stable id for this device/tab. |
| `collectManifest(modules, { schemaVersion? })` | The codegen-free client manifest (server twin of `collectTables`). |

Also exported: `SyncStatus`, `RecoveryStatus`, `LocalStore`, `SyncTransport`,
`LocalFirstManifest`, `ScopeDefinition`, `SyncScope`, `LocalCommit`,
`MutationStatus`, `AttachmentBackend`.

```ts
type SyncStatus = {
  online: boolean;
  syncing: boolean;
  pendingMutations: number;
  lastPushAt: number | null;
  lastPullAt: number | null;
  lastError: string | null;
  blockedBySchemaMismatch: boolean;
  partial: boolean;
  recovery: RecoveryStatus;   // { rejectedOperations, olderSchemaOperations, failedAttachments, failedGroups }
};
```

## `convex-localfirst/testing`

```ts
function createTestHarness<Modules>(options: {
  modules: Modules;
  schemaVersion?: number;
  access?: SyncConfig["access"];
  onWrite?: SyncConfig["onWrite"];
  serverStamp?: Record<string, { fields; stamp }>;
  userId?: string | null;
  clientId?: string;
  now?: number;                 // starting logical clock, default 1_000_000
}): {
  db;                           // typed createLocalDb root
  engine;                       // current engine (swaps on setUser)
  Provider;                     // publishes the engine to the local-first hooks
  server: { seed(table, rows); rows(table); ledger(); store };
  goOffline(); goOnline(); settled();
  clock: { now(); advance(ms) };
  setUser(userId); userId(); dispose();
}
```

Runs the REAL `handlePush`/`handlePull` behind a controllable transport and a real
engine. Also exports `MemoryServerStore`. See [Testing](/docs/testing).

## `convex-localfirst/component`

A mountable Convex component holding all sync bookkeeping (ledger, change log, id
map, row versions, presence). Mount with `app.use(localfirst)` from
`convex-localfirst/component/convex.config.js`; the parent app calls its public
functions via `components.convexLocalFirst.*`. See [Syncing](/docs/syncing#the-component).

## `npx convex-localfirst` — the CLI

`init` scaffolds a complete starter; `check` fails if any Convex function writes a
local-first table directly with `ctx.db`. See [The CLI](/docs/cli).
