Everything below is the public surface. Engine internals (rebase, the engine
class, the store implementation details) are intentionally not exported.

## `convex-localfirst/react`

### `ConvexProvider`

```ts
function ConvexProvider(props: {
  client: ConvexReactClient;
  children: React.ReactNode;
  localFirst?: LocalFirstProviderConfig;
}): JSX.Element
```

Drop-in for Convex's provider. Without `localFirst`, behaves exactly like
`convex/react`. See [the provider](/docs/react-hooks#the-provider).

### `LocalFirstProviderConfig`

```ts
type LocalFirstProviderConfig = {
  modules?: Record<string, unknown>; // imported lf.table modules (runtime manifest, no codegen)
  schemaVersion?: number;            // gates sync on incompatible shape changes; default 1
  manifest?: LocalFirstManifest;     // escape hatch / tests, instead of modules
  userId?: string | null;
  clientId?: string;
  store?: LocalStore;                // default: IndexedDB in the browser (namespaced by userId)
  databaseName?: string;             // default "convex-localfirst"
  namespace?: string;                // default userId
  sync?: { push?: FunctionReference<"mutation">; pull?: FunctionReference<"query"> };
                                     // default api.sync.push / api.sync.pull
  transport?: SyncTransport;         // escape hatch: bring your own
  nameOf?: FunctionNameResolver;     // defaults to Convex getFunctionName
};
```

### `useQuery`

Args and result are inferred from the function reference — no explicit generics.

```ts
function useQuery<Query extends FunctionReference<"query">>(
  reference: Query,
  args?: FunctionArgs<Query> | "skip",
  options?: { initial?: FunctionReturnType<Query>; sync?: "auto" | "off" }
): FunctionReturnType<Query> | undefined
```

### `useMutation`

```ts
function useMutation<Mutation extends FunctionReference<"mutation">>(
  reference: Mutation
): (args: FunctionArgs<Mutation>) => LocalFirstMutationCall<FunctionReturnType<Mutation>>
```

### `LocalFirstMutationCall`

```ts
type LocalFirstMutationCall<T> = Promise<T> & {
  opId: string;
  local: Promise<LocalCommit>;
  server: Promise<T>;
  status: () => MutationStatus;
};
```

### `useSyncStatus`

```ts
function useSyncStatus(): SyncStatus
```

`ConvexReactClient`, `Authenticated`, `Unauthenticated`, `AuthLoading`,

## `convex-localfirst/core`

### `IndexedDbStore`

```ts
new IndexedDbStore({ databaseName: string; namespace: string })
```

Browser persistence. `namespace` isolates per-user data on a shared device
([invariant I9](/docs/reference/invariants)).

### `MemoryLocalStore`

In-memory `LocalStore` — the default when no `store` is given (tests, SSR).

### `createConvexTransport`

```ts
function createConvexTransport(options: {
  client: ConvexReactClient;
  push: unknown;     // api.sync.push
  pull: unknown;     // api.sync.pull
  clientId: string;
  userId: string;
}): SyncTransport
```

### `createClientId`

```ts
function createClientId(): string
```

A stable id for this device/tab.

### `SyncStatus`

```ts
type SyncStatus = {
  online: boolean;
  syncing: boolean;
  pendingMutations: number;
  lastPushAt: number | null;
  lastPullAt: number | null;
  lastError: string | null;
  blockedBySchemaMismatch: boolean;
  partial: boolean; // still catching up to the server (a large cold start drained past the per-pull cap)
};
```

### `collectManifest`

```ts
function collectManifest(
  modules: Record<string, unknown>,        // imported lf.table modules
  options?: { schemaVersion?: number }
): LocalFirstManifest
```

The codegen-free client manifest: reads the declarations (and their original
closures) off the imported `lf.table` modules at runtime. The provider's
`modules` option calls this for you; use it directly for a headless engine.
Its server twin is [`collectTables`](#collecttables).

Also exported: `LocalStore`, `SyncTransport`, `LocalFirstManifest`,
`FunctionNameResolver`, `LocalCommit`, `MutationStatus`, `ScopeDefinition`,
and `SyncScope`.

## `convex-localfirst/server`

### `createLocalFirst`

```ts
function createLocalFirst<Ctx>(options?: {
  defaults?: { idField?: string };
  schemaVersion?: number;
}): LfFactory
```

The authenticated user is resolved server-side at sync time (in
`createSyncFunctions`, from `ctx.auth`), not configured here.

Returns the `lf` factory:

| Member | Signature |
| --- | --- |
| `lf.table(name, options)` | `{ shape; scope; idField?; timestamps?; indexes?; setFields?; counterFields? }` → `{ table, query, insert, patch, remove }` |
| `lf.byUser(field)` | `ScopeDefinition` |
| `lf.byWorkspace({ workspaceIdField, membershipTable })` | `ScopeDefinition` |
| `lf.byProject({ projectIdField, membershipTable })` | `ScopeDefinition` |

The builder specs:

```ts
table.query({  args, index, key, order?, initial? })
table.insert({ args, value })   // value: ({ ctx, auth, args, now, localId }) => Row
table.patch({  args, id, patch })
table.remove({ args, id })
```

Every builder's spec is optional — a bare call derives it from the shape:
`insert()` takes every field except the stamped ones (a `byUser` owner comes
from auth, timestamps stamp automatically), `patch()` takes `id` plus any
subset of non-partition fields, `remove()` takes `id`.

### `createSyncFunctions`

The turnkey server sync surface — builds the `push` mutation + `pull` query.

```ts
function createSyncFunctions(options: {
  component: typeof components.convexLocalFirst;
  mutation: MutationBuilder;   // from _generated/server
  query: QueryBuilder;         // from _generated/server
  tables: SyncConfig["tables"];
  internalMutation?: InternalMutationBuilder; // exposes `gc` as an internal cron target
  // Required when a table is byWorkspace/byProject. `member` returns a role (any value)
  // or null/undefined to deny; `read`/`write` are optional per-row filters.
  access?: {
    member: (ctx, info: { userId; scopeValue; table; membershipTable? }) => Role | null | Promise<Role | null>;
    read?: (ctx, info: { userId; role; table; row }) => boolean | Promise<boolean>;
    write?: (ctx, info: { userId; role; table; action; before; patch?; proposed }) => boolean | Promise<boolean>;
  };
  getUserId?: (ctx) => string | null | Promise<string | null>; // default: identity.tokenIdentifier
  schemaVersion?: number;
  now?: () => number;
  changeRetentionMs?: number;  // change-log GC horizon, default 30 days
  serverStamp?: Record<table, { fields: string[]; stamp: (ctx, { userId, value }) => Record<string, unknown> | undefined }>;
  onWrite?: (ctx, { table, action, before, after, userId, functionName }) => Promise<void>;
}): { push; pull; presence; presenceList; gc; serverWriter }
```

See [Syncing](/docs/syncing#createsyncfunctions).

`serverWriter(ctx)` is the trusted server-side writer: call it from your own
mutations (activity fan-out, importers, crons) to insert/patch/remove rows in
local-first tables **through the change log**, so every client syncs the write
like any other change:

```ts
const w = serverWriter(ctx);
const { localId } = await w.insert("activities", { workspaceId, verb: "created" });
await w.patch("issues", issueId, { sub_issues_count: 3 });
await w.remove("comments", commentId);
```

### `serverSync` engine (advanced)

For full control you can skip `createSyncFunctions` and drive the pure engine:

```ts
function handlePush(store: ServerStore, config: SyncConfig, request): Promise<PushResult>
function handlePull(store: ServerStore, config: SyncConfig, request): Promise<PullResult>
```

Plus the types `ServerStore`, `ServerOperation`, `StoredChange`, `SyncConfig`.

## `convex-localfirst/component`

A mountable Convex component. Mount with `app.use(localfirst)` from
`convex-localfirst/component/convex.config.js`; call its public functions via
`components.convexLocalFirst.*` (`ops`, `changes`, `idMaps`, `presence`). See
[Syncing](/docs/syncing#the-component).

## `convex-localfirst/yjs`

Wire a Yjs CRDT (rich text, nested lists) onto the local-first log. `react` and `yjs`
are peer deps. See [Document mode](/docs/document-mode).

```ts
function useCollaborativeDoc(options: {
  docId: string;                                  // must uniquely identify the doc across scopes
  updates: ReadonlyArray<Record<string, unknown>>; // live rows in scope (filtered to docId internally)
  append: (updateBase64: string) => MutationLike; // persist a local edit as a row
  prune?: (rowId: string) => MutationLike;        // enables compaction; omit to disable
  idField?: string;                                // per-row id field (default "_id")
  docField?: string;                               // row field naming the doc (default "doc")
  updateField?: string;                            // row field holding the update (default "update")
  compaction?: CompactionOptions | false;          // cadence; default on when `prune` is provided
}): { doc: Y.Doc; status: DocStatus }             // status: { synced, pendingUpdates, lastError, compacting }

// Cursor/selection awareness lives at the `convex-localfirst/yjs/awareness` subpath:
//   useDocAwareness(doc, { docId, scope, state }) → { awareness, peers }

// codec helpers (framework-agnostic)
function bytesToBase64(bytes: Uint8Array): string;
function base64ToBytes(b64: string): Uint8Array;
function makeSnapshot(doc: Y.Doc): string;
function applyUpdateSafe(doc: Y.Doc, base64: string, origin: unknown): boolean;
const REMOTE_ORIGIN: unique symbol;
```
