## Invariants

These hold by construction — they're the contract the framework is built to keep.

| # | Invariant |
| --- | --- |
| **I1** | The live view is **derived**: `deriveView(canonical + replay(pending))`. Server changes never clobber un-synced local ops. |
| **I4** | Operations have a deterministic order, so replays are stable (even within one millisecond). |
| **I5** | `applyServerChange` is version-guarded — out-of-order or stale changes are safe. |
| **I6** | No "sync the whole database" mode is reachable in production. Every table is explicitly scoped. |
| **I7** | The **client never decides authorization.** The server enforces ownership (`byUser`) and membership (`byWorkspace` / `byProject`) on push *and* pull. |
| **I8** | Side-effecting / server functions are **server-only by default.** A function is local-first only if declared via `lf.table` and present in the manifest. |
| **I9** | Logout clears or namespaces local data so the next user can't read it. |
| **I10** | `check` catches direct `ctx.db` writes to local-first tables (which would bypass the ledger and change log). |
| **I11** | Ordinary Convex functions keep working through the same hooks (fallback). |

## Non-negotiable constraints

- Do **not** run a Convex backend in the browser.
- Do **not** promise arbitrary Convex functions become local-first automatically.
- Do **not** synchronize an entire application database by default.
- Convex remains authoritative for validation, auth, idempotency, and canonical
  ordering.
- Local-first tables require explicit scope declarations.
- Writes to local-first tables must pass through the DSL or generated wrappers.

## Deferred (bounded, not faked)

These are intentionally out of scope for now, with a clear seam to add them:

- **Server-side mutation re-execution.** The server applies the client-computed
  insert value / patch after its authz + schema checks; it does not re-run your
  mutation body. Encode hard invariants as Convex schema validators for now — see
  [Syncing → Trust model](/docs/syncing#trust-model--current-limitations).
- **Automerge rich-text.** Yjs already ships (`convex-localfirst/yjs`); Automerge
  rides the same append-only-log approach (one binary change per row) later.
- **A full Next.js plugin.** The Vite alias strategy is implemented and
  documented; the Next variant is not yet shipped.
- **Batched component writes** — one component mutation per push instead of a few
  calls per op, for high-throughput backends.

## Shipped

- **Typed client + declared relations** — `createLocalDb(modules)` and
  `.with(...)` derive rows, scopes, and relations from the schema. See
  [Typed client](/docs/typed-db).
- **Serializable filter AST + saved views** — `.filter(spec)`, `serializeFilter` /
  `parseFilter`. See [Filters](/docs/filters).
- **Group-by, live counts & ranking** — `.groupBy()`, `useLiveCounts`, and the
  fractional-rank helpers. See [Grouping](/docs/grouping).
- **Incremental query engine** — local secondary indexes; subscribed queries
  update by single-row deltas instead of re-scanning.
- **Atomic write groups** — `useBatch()` lands or rejects a group as one
  offline-durable unit. See [Batches](/docs/batches).
- **Permission-aware UI** — the synced role plus a client-side write mirror
  (`useRole` / `useCan`, `clientCan`). See [Permissions](/docs/permissions).
- **Undo / redo** — scope-aware inverse-op stacks. See [Undo](/docs/undo).
- **Full-text search** — incremental, memory-resident `useSearch` over declared
  `searchFields`. See [Search](/docs/search).
- **Offline attachments** — `createAttachmentFunctions` + `useCreateAttachment` /
  `useAttachmentUpload`. See [Attachments](/docs/attachments).
- **Devtools** — `<LocalFirstDevtools />`, a dev-only inspector. See [Devtools](/docs/devtools).
- **Test harness** — `createTestHarness` (`convex-localfirst/testing`). See [Testing](/docs/testing).
- **Presence & awareness** — `usePresence` and `useDocAwareness` (cursor/selection
  presence). See [Presence](/docs/presence) and [Document mode](/docs/document-mode).
- **Yjs rich text** (`convex-localfirst/yjs`) — collaborative documents over the
  local-first log (`useCollaborativeDoc`). See [Document mode](/docs/document-mode).
- **Convergent merges** — opt-in `setFields` / `counterFields`. See
  [Defining tables](/docs/defining-tables#merges).
- **Server-authored writes** — `serverWriter(ctx)` writes local-first tables
  through the change log from ordinary mutations.
- **Snapshot bootstrap + change-log GC** — cold clients load current rows instead
  of replaying history; delivered changes are pruned after `changeRetentionMs`.
- **`createSyncFunctions`** — the server `sync.ts` is a dozen lines, not ~150.
- **Full type inference** — `useQuery` / `useMutation` infer args and result from
  the function reference; no explicit generics.
