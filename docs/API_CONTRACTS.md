# API Contracts

## Core runtime contract

The core runtime must be framework agnostic. React, CLI, and server packages must depend on the core package, not the reverse.

## Store contract

A store must provide:

- row reads
- canonical row reads
- local view writes
- canonical writes
- outbox persistence
- cursor persistence
- subscription notifications
- namespace isolation
- atomic transactions when supported

## Transport contract

A transport must provide:

- `push(request)`
- `pull(request)`
- optional `poke(scope)`
- optional `status()`

The default Convex transport is generated by codegen and calls Convex functions.

## Manifest contract

The generated manifest must describe:

- tables
- scopes
- queries
- mutations
- indexes
- conflict policies
- sync functions
- schema version

The manifest must not include secrets.

## Server DSL contract

The server DSL must generate normal top-level Convex functions. It must also expose metadata for codegen. Runtime metadata must not affect Convex security.

## React contract

The React wrapper must preserve normal Convex behavior for normal Convex functions.

The wrapper may add capabilities for local-first functions, but must not break code that already awaits a Convex mutation promise.
